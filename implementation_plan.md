# Implementing an AI Personal Career Agent

We are transitioning the generic feature-based architecture into a specific product: an **AI Personal Career Agent**.
To support this, the application UI will utilize a global Navbar, and a conditional Sidebar for navigating the core authenticated features.

## Minimal MVP Pages Needed

For an AI Career Agent, you need the following minimal pages to deliver end-to-end value:

1. **Landing / Authentication (`/auth`)**
   - Splash page explaining the value proposition.
   - Magic Link or OAuth Login form.
   - *Layout:* Navbar only.

2. **Dashboard / Overview (`/dashboard`)**
   - The main hub. Displays a summary of your career progress, active job applications, and quick actions ("Analyze Resume", "Prep for Interview").
   - *Layout:* Navbar + Sidebar.

3. **Resume Builder & Analyzer (`/resume`)**
   - Where users upload their current resume, and the AI suggests improvements or tailors it to specific job descriptions.
   - *Layout:* Navbar + Sidebar.

4. **Job Tracker (`/jobs`)**
   - A Kanban board or list view to track jobs you are applying to, storing the job description for the AI to reference.
   - *Layout:* Navbar + Sidebar.

5. **AI Interview Prep (`/interview`)**
   - A chat/interactive interface where the AI conducts a mock interview based on a saved job description and resume.
   - *Layout:* Navbar + Sidebar.

## Minimal Database Schema (Future Step)

To support the above MVP features, you will need a relational database to store the user's career artifacts. Supabase will act as the backend.

### 1. `profiles`
Extends the native `auth.users` with app-specific logic.
- `id` (uuid, PK, references auth.users)
- `first_name` (text)
- `last_name` (text)
- `career_goal` (text) - e.g. "Senior Frontend Engineer"
- `created_at` (timestamp)

### 2. `resumes`
Stores the user's base resume and any tailored versions generated by AI.
- `id` (uuid, PK)
- `user_id` (uuid, references profiles)
- `title` (text) - e.g. "Software Engineer Base", "Google Tailored"
- `content` (jsonb / text) - The actual resume payload
- `is_base_resume` (boolean)
- `created_at` (timestamp)

### 3. `jobs`
The Job Tracker. Stores external job descriptions crucial for AI context generation.
- `id` (uuid, PK)
- `user_id` (uuid, references profiles)
- `company_name` (text)
- `role_title` (text)
- `job_description` (text) - Used by AI for Interview Prep & Resume Tailoring!
- `status` (enum: 'saved', 'applied', 'interviewing', 'rejected', 'offer')
- `linked_resume_id` (uuid, references resumes, nullable) - Which resume was used for this
- `created_at` (timestamp)

### 4. `interview_sessions`
Stores chat history and AI feedback for mock interviews.
- `id` (uuid, PK)
- `user_id` (uuid, references profiles)
- `job_id` (uuid, references jobs) - The context the AI is interviewing for
- `chat_history` (jsonb) - The back-and-forth Q&A array
- `ai_feedback` (text) - The final score/feedback generated post-interview
- `created_at` (timestamp)
